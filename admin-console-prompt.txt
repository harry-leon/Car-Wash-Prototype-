Đọc kỹ cấu trúc project hiện tại trước khi bắt đầu để đảm bảo đúng conventions.

Tạo toàn bộ Admin Console module theo đúng yêu cầu sau:

=== RÀNG BUỘC QUAN TRỌNG ===
- Chỉ tạo/sửa file trong src/modules/admin-console/
- KHÔNG sửa bất kỳ file nào ngoài folder đó (không đụng App.tsx, router config, hay bất kỳ file nào khác)
- Toàn bộ dùng mock data, không có API call
- Dùng shadcn/ui components có sẵn trong project
- Dùng Tailwind CSS + TypeScript

=== TASK 4.1 — Admin Dashboard ===
Route: /admin/dashboard
Hiển thị 5 KPI cards:
- Today bookings
- Completed washes
- No-show count
- Points issued
- Active promotions
Mỗi card có: label, value, delta so hôm qua, icon, màu riêng.
Bên dưới có bảng Recent Bookings mock.

=== TASK 4.2 — All Bookings Page ===
Route: /admin/bookings
Bảng có các cột: Booking code, Customer name, Vehicle plate, Service package, Scheduled time, Status, Action.
Filter mock theo: Status, Date, Customer name (dùng local state).
Action cho phép admin đổi status bằng useState:
CONFIRMED, CHECKED_IN, IN_PROGRESS, COMPLETED, CANCELLED, NO_SHOW

=== TASK 4.3 — Customers List Page ===
Route: /admin/customers
Bảng có các cột: Customer name, Email, Phone, Tier, Available points, Lifetime points, Role, Status, Action (link đến detail).
Tier: MEMBER, SILVER, GOLD, DIAMOND
Role: CUSTOMER, ADMIN, STAFF
Status: ACTIVE, SUSPENDED

=== TASK 4.4 — Customer Detail Page ===
Route: /admin/customers/:id
Tab layout gồm đúng 6 tabs: Profile, Vehicles, Bookings, Wash History, Point Transactions, Tier History.
Trong tab Profile, admin có thể mock đổi:
- Role: CUSTOMER, ADMIN, STAFF
- Status: ACTIVE, SUSPENDED
Dùng local state, không cần lưu thật.

=== TASK 4.5 — Loyalty Page ===
Route: /admin/loyalty
Có đúng 3 phần: Tier Rules, Tier History, Points Audit.
KHÔNG làm Manual Point Adjustment.
Tier Rules hiển thị 4 tiers với minLifetimePoints:
- Member: 0
- Silver: 5,000
- Gold: 15,000
- Diamond: 40,000
Points Audit hiển thị transaction với đủ các cột:
Customer, Booking code, Type, Amount, Available balance after, Lifetime balance after, Created time.

=== TASK 4.6 — Promotions Page ===
Route: /admin/promotions
Chức năng: Promotion list + Create + Edit + Deactivate (tất cả bằng useState mock).
Form dùng Dialog từ shadcn/ui.
Promotion type gồm đúng 3 loại:
- Apply to all members
- Apply to selected tiers
- New customers only
Hiển thị các cột: Name, Type, Target tiers, Discount percent, Start date, End date, Active/inactive, Usage count.

=== TASK 4.7 — Reports Page ===
Route: /admin/reports
Hiển thị mock data cho:
- Daily/monthly statistics
- Booking trends (dùng Recharts BarChart, đã có trong project)
- No-show rate
- Promotion usage
- Point transaction summary
- Promotion effectiveness
Phần còn lại dùng card hoặc table đơn giản, không bắt buộc chart phức tạp.

=== TASK 4.8 — Settings Page ===
Route: /admin/settings
Form mock với useState cho các phần:
- Business information
- Cancellation policy
- Point conversion rate
- No-show threshold
- Supported languages
- Currency format
Không cần lưu thật, chỉ cần local state.

=== MOCK DATA YÊU CẦU ===
- Tên khách hàng tiếng Việt (Nguyễn Văn An, Trần Thị Bảo, v.v.)
- Biển số xe format: 51A-123.45
- Ít nhất 10 customers, 13+ bookings
- Đủ dữ liệu để test filter, tabs, và tất cả các trạng thái

Export adminRoutes object chứa tất cả page components.
KHÔNG tự đăng ký vào root router, chỉ export để team member khác dùng:
